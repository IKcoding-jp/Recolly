# PR前の理解度チェックリスト

## 必須チェック
- [ ] このPRで変更したコードの動作を、自分の言葉で説明できるか
- [ ] 新しく導入したライブラリ・パターンの役割を理解しているか
- [ ] このコードが壊れやすいポイントを把握しているか
- [ ] 理解が追いついていない箇所に `// TODO: 理解する` コメントを入れたか

## 記録チェック
- [ ] 設計判断があった場合、ADRを作成または更新したか
- [ ] 新技術を使った場合、学習ノートの作成を検討したか

## AI利用の記録
- [ ] このPRにAI生成コードを含むか
- [ ] 含む場合、特に注意が必要な箇所を把握しているか
