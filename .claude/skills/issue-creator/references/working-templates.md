# Working Documents テンプレート

## context.md（旧 requirement.md）

コンテキスト復元が主目的。仕様書を複製せず、参照リンク + Issue固有の補足に留める。

```markdown
# コンテキスト: [タイトル]

Issue: #XX

## 仕様書参照
- [メイン仕様書](../../superpowers/specs/YYYY-MM-DD-xxx.md)（該当セクション名を明記）
- [ADR-XXXX](../../adr/XXXX-xxx.md)（関連する設計判断があれば）

## このIssueで実現すること
[仕様書の中でこのIssueがカバーする範囲を1〜3文で]

## このIssue固有の補足
[仕様書に書かれていないが、ブレインストーミングで決まったこと・制約・注意点]

## スコープ外
- 今回は対応しないこと
```

## tasklist.md

```markdown
# タスクリスト: [タイトル]

Issue: #XX

## フェーズ1: [準備/調査]
- [ ] タスク1
- [ ] タスク2

## フェーズ2: [実装]
- [ ] タスク3
- [ ] タスク4

## フェーズ3: [テスト/確認]
- [ ] タスク5
- [ ] タスク6

## 完了条件
- 条件1
- 条件2
```

## design.md

```markdown
# 設計書: [タイトル]

Issue: #XX

## 変更対象ファイル
| ファイル | 変更内容 |
|---------|---------|
| `path/to/file` | 概要 |

## 実装方針
[どう実装するか]

## 影響範囲
[この変更で影響を受ける箇所]

## 関連ADR
- [ADR-XXXX](../../adr/XXXX-xxx.md)（あれば）
```
