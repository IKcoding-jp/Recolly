# プロフィール編集機能 — 設計スペック

**Issue:** #89
**日付:** 2026-04-01
**スコープ:** アバター画像編集、自己紹介（bio）編集、お気に入りベスト5

## 概要

プロフィールページ（`/users/:id`）上で、自分のプロフィール情報を直接編集できる機能を追加する。

### スコープ

1. **アバター画像のアップロード** — ホバーでカメラアイコン → 画像変更
2. **自己紹介（bio）の編集** — インライン編集（100文字制限）
3. **お気に入りベスト5の設定・表示** — ライブラリから選択、ランキング/お気に入りモード切り替え

### スコープ外

- 記録の公開/非公開切り替え（当初のIssue記載から削除）

## データモデル

### 既存テーブルの変更

**users テーブル**

| カラム | 型 | 変更内容 |
|--------|------|---------|
| bio | text | 既存。バリデーション追加（最大100文字） |
| avatar_url | string | 既存。変更なし |
| favorite_display_mode | string | **新規追加**。デフォルト `"ranking"`。値: `"ranking"` / `"favorites"` |

### 新規テーブル

**favorite_works テーブル**

| カラム | 型 | 説明 |
|--------|------|------|
| id | bigint (PK) | |
| user_id | bigint (FK → users) | NOT NULL |
| work_id | bigint (FK → works) | NOT NULL |
| position | integer | 並び順 1〜5。NOT NULL |
| created_at | datetime | |
| updated_at | datetime | |

**制約:**

- `user_id` + `work_id` でユニーク（同じ作品を2回登録不可）
- `user_id` + `position` でユニーク（同じ順位に2作品不可）
- `position` は 1〜5 の範囲

**アソシエーション:**

- `User has_many :favorite_works, dependent: :destroy`
- `FavoriteWork belongs_to :user`
- `FavoriteWork belongs_to :work`

## API エンドポイント

### プロフィール更新

**`PATCH /api/v1/profile`** — 自分のプロフィールを更新（認証必須）

リクエスト:
```json
{
  "bio": "アニメと映画が好きです。",
  "avatar_url": "uploads/avatars/xxx.jpg",
  "favorite_display_mode": "ranking"
}
```

- 送ったフィールドだけ更新される（部分更新）
- レスポンス: 更新後のユーザー情報（200 OK）

### お気に入り作品取得

**`GET /api/v1/users/:id/favorite_works`** — ユーザーのベスト5を取得（認証不要）

レスポンス:
```json
{
  "favorite_works": [
    { "id": 1, "position": 1, "work": { "id": 10, "title": "進撃の巨人", "cover_image_url": "..." } },
    { "id": 2, "position": 2, "work": { "id": 20, "title": "君の名は。", "cover_image_url": "..." } }
  ],
  "display_mode": "ranking"
}
```

### お気に入り作品一括更新

**`PUT /api/v1/profile/favorite_works`** — ベスト5を一括更新（認証必須）

リクエスト:
```json
{
  "favorite_works": [
    { "work_id": 10, "position": 1 },
    { "work_id": 20, "position": 2 },
    { "work_id": 30, "position": 3 }
  ]
}
```

- 個別の追加/削除APIではなく、「現在のベスト5はこれ」と一括送信する方式
- 理由: 並び替えや追加/削除を個別APIで扱うと状態管理が複雑になるため
- レスポンス: 更新後のfavorite_works（200 OK）

## フロントエンド UI・インタラクション

### 1. アバター画像の編集

- **通常時:** 既存の丸型アバター（88px、イニシャルまたは画像）
- **ホバー時（自分のページのみ）:** アバターが暗転 + カメラアイコンのオーバーレイ表示
- **クリック:** ファイル選択ダイアログが開く → S3直接アップロード → `PATCH /api/v1/profile` で `avatar_url` を更新
- **アップロードフロー:** 既存の `ImageUploader` コンポーネントのUI/UXを活用するが、アバターは `Image` モデル（Work用）ではなく `User.avatar_url` に直接保存する。presign エンドポイントは `avatars/` プレフィックスのS3キーを発行する
- **制約:** JPEG, PNG, GIF, WebP / 最大10MB
- **画像削除:** イニシャルアイコンに戻る

### 2. 自己紹介（bio）のインライン編集

**表示位置:** 利用開始日のすぐ下

- **通常表示（bioあり）:** bioテキスト + ✏️アイコン（自分のページのみ）
- **通常表示（bioなし・自分のページ）:** 「＋ 自己紹介を追加」プレースホルダー
- **通常表示（bioなし・他人のページ）:** セクション非表示
- **編集モード:** テキストエリアに切り替わり、文字数カウンター（n / 100）+ 保存/キャンセルボタン
- **文字数制限:** 100文字。`maxLength` で入力制限
- **空文字保存:** bio未設定状態に戻る

### 3. お気に入りベスト5

**表示位置:** プロフィールヘッダー直下、統計カードの上

**モード切り替え:**
- セクションタイトル横にトグルスイッチ（「ランキング ↔ お気に入り」）
- 自分のページのみ表示。トグル切り替えは即座にAPIで保存
- 切り替えても作品の並び順データは保持される

**ランキングモード:**
- 順位バッジ付き（1位=金、2位=銀、3位=銅、4-5位=グレー）。色は `tokens.css` にCSS変数として定義する
- 1位のみカバー画像に金枠

**お気に入りモード:**
- バッジ・枠なしでシンプルにカバー画像を並べる

**作品の追加（自分のページのみ）:**
- 空きスロットに「＋」ボタン → 作品選択モーダルが開く
- 5枠未満でも保存可（1〜4作品OK）

**未設定時:**
- 自分のページ: 案内テキスト + 「＋ 作品を追加」ボタン
- 他人のページ: セクション非表示

**他人のページでの表示:**
- 編集UI（トグル、＋ボタン）は一切非表示
- 設定済みの内容を閲覧モードで表示

### 4. 作品選択モーダル

**構成:**
- 検索バー（テキスト入力で作品名を絞り込み）
- ジャンルフィルタータブ（PublicLibrary の `filterButton` パターンを再利用）
- 作品グリッド（カバー画像 + タイトル）

**挙動:**
- ライブラリに記録済みの作品のみ表示
- 既にベスト5に選ばれている作品はグレーアウト（選択不可）
- 作品クリックで選択 → モーダルが閉じて反映
- ライブラリが空の場合: 「ライブラリに作品を追加してから設定してください」と案内

## エラーハンドリング・エッジケース

| ケース | 挙動 |
|--------|------|
| アバターアップロード失敗 | エラーメッセージ表示 + 再試行リンク（既存 ImageUploader 挙動） |
| bio 100文字超過 | `maxLength` で入力制限。カウンターでリアルタイム表示 |
| bio 保存失敗 | フィールド下にエラーメッセージ |
| 同じ作品を2回選択 | モーダルでグレーアウト表示（選択不可） |
| ベスト5に選んだ作品がライブラリから削除された | `dependent: :destroy` で `favorite_works` も自動削除 |
| ライブラリが空でベスト5を設定しようとした | モーダル内に案内テキスト表示 |

## 既存パターンとの一貫性

### スタイル方針

- CSS Modules（`.module.css`）を使用
- 全てのスタイル値は `tokens.css` の変数を使用（ハードコード禁止）
- 見出し: `--font-heading` (Fraunces)、本文: `--font-body` (Zen Kaku Gothic New)
- ボタン: 既存 `Button` コンポーネント（保存=primary、キャンセル=secondary）
- アニメーション: `--transition-fast` (150ms) / `--transition-normal` (250ms) のみ
- レスポンシブ: `768px` ブレークポイント

### 新規コンポーネント

1. **ToggleSwitch** — ランキング/お気に入り切り替え用。`--color-text`（黒）をアクティブ色にする
2. **FavoriteWorkSelector（モーダル）** — 作品選択用。既存モーダルパターン（overlay: `rgba(0,0,0,0.5)` + `z-index: 100`、本体: `max-width` + `width: 90%` + `border-radius: 8px`）を踏襲

### 既存コンポーネントの再利用

| コンポーネント | 用途 |
|---------------|------|
| `ImageUploader` | アバター画像アップロード |
| `Button` | 保存・キャンセル・追加ボタン |
| `Typography` | セクション見出し |
| `GENRE_FILTERS` | 作品選択モーダルのフィルター定義 |

### UserProfileHeader の拡張

- `isOwner: boolean` props を追加（自分のページかどうか）
- `onAvatarChange`、`onBioSave` 等のコールバック props を追加
- 自分のページ閲覧時のみ編集UIを表示

### エラー表示

既存パターンに従う:
- `color: var(--color-error)` (#c0392b)
- `font-size: var(--font-size-meta)`

### ローディング

- アバターアップロード中: `ImageUploader` 既存のプログレス表示
- ベスト5保存中: ボタンを `disabled` + テキスト「保存中...」
