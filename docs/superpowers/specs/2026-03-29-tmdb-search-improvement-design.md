# TMDB検索改善 設計書

## 背景

ドラマ検索（TMDB）が他ジャンルに比べて検索精度・件数ともに劣る問題がある。

### 問題1: 中黒なしで0件になる

TMDBでは外来語タイトルに中黒「・」が含まれる（例: 「ウォーキング・デッド」）。
ユーザーは中黒なしで検索する（例: 「ウォーキングデッド」）ため、マッチせず0件になる。

**再現ケース:**
- 「ウォーキングデッド」→ 0件（正式名: ウォーキング・デッド）
- 「ゲームオブスローンズ」→ 0件（正式名: ゲーム・オブ・スローンズ）
- 「ペーパーハウス」→ 0件（正式名: ペーパー・ハウス）
- 「ブレイキングバッド」→ 0件（正式名: ブレイキング・バッド）

### 問題2: 件数が少ない

現在 `search/multi`（映画+TV+人物の横断検索）を使用しており、1リクエスト20件が上限。
映画とTVが混在して枠を消費し合うため、ドラマ検索時にTV番組の結果が少なくなる。

## 設計

### 変更1: API切り替え（search/tv, search/movie への分離）

`search/multi` を廃止し、ジャンル専用のエンドポイントに切り替える。

| 検索ジャンル | 現在のAPI | 変更後のAPI |
|---|---|---|
| ドラマ | search/multi → tvのみフィルタ | **search/tv** |
| 映画 | search/multi → movieのみフィルタ | **search/movie** |
| すべて（media_type指定なし） | search/multi | **search/movie + search/tv を並列呼び出し** |

**効果:**
- ドラマ検索時にTV番組だけで20件取得できる（映画が枠を消費しない）
- 映画検索時も同様に映画だけで20件取得できる

**注意:**
- `fetch_japanese_description`（AniList説明補完用）は既存の `search/multi` のまま残す（検索結果には表示しないため影響なし）
- 日本アニメ除外ロジック（`japanese_animation?`）は `search/movie` 使用時のみ適用（TVアニメはAniList側で取得するため `search/tv` でも除外が必要）

### 変更2: 中黒バリエーション検索

長音「ー」の直後にカタカナが続く場合、そこに中黒「・」を挿入した候補を生成し、追加検索する。

**分割ルール:**
- 正規表現: 長音「ー」の直後にカタカナ文字（「ァ-ヶ」）が続く箇所で中黒を挿入
- 例: `ウォーキングデッド` → `ー` の後に `デ` → `ウォーキング・デッド`

**生成例:**

| 入力 | 生成結果 |
|---|---|
| ウォーキングデッド | ウォーキング・デッド |
| ペーパーハウス | ペーパー・ハウス |
| ブレイキングバッド | ブレイキング・バッド |
| ストレンジャーシングス | ストレンジャー・シングス |
| ゲームオブスローンズ | ゲーム・オブスローンズ（部分的。TMDBの部分一致に期待） |

**実行条件:**
- 元のクエリでの検索結果が3件以下の場合のみ追加検索を実行
- 中黒挿入版のクエリが元と同じ場合（変化なし）はスキップ
- 結果はTMDB IDで重複除去してマージ

### 変更箇所

`backend/app/services/external_apis/tmdb_adapter.rb` のみ。

### テスト方針

- 既存テストの `search/multi` モックを `search/tv` / `search/movie` に変更
- 中黒バリエーション検索のテストを追加
  - 結果が少ない場合に中黒挿入版で追加検索されること
  - 結果が十分な場合は追加検索されないこと
  - 重複が除去されること
