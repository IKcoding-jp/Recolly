## 1. Product Overview

- **ワンライナー**: アニメ・映画・本・漫画・ゲームを**ジャンルの壁を越えて**記録できる、物語好きのためのライブラリ
- **何をするか**: 複数ジャンルのコンテンツを 1 つのアプリで記録・進捗管理・振り返りできる。外部 API 連携で作品検索は即座、視聴/読書状況はワンクリックで更新
- **カテゴリ（棚）**: 作品ログ管理アプリ / 読書&視聴記録サービス / メディアトラッカー
- **プロダクトタイプ**: BtoC Web アプリ（SaaS 型、PWA 対応済み）
- **ビジネスモデル**: 現状無料。**永久無料 + 付加価値のみ有料化** の方針を将来ランディングに明記予定

## 2. Target Audience

### ビーチヘッド（最初の獲得対象）

- **マルチアプリ疲労派**: 既に Annict・Filmarks・読書メーター・Steam 等を併用していて、切り替え疲れを感じている人
- **実在確認**: IK さん自身がこのペルソナに該当するため、自己検証可能

### 広がり先（ビーチヘッド獲得後）

1. 記録マニア派（読書メーター歴長め、他ジャンルも同粒度で記録したい）
2. 原作横断派（原作本 → アニメ → 実写映画 と横断消費）
3. ライト多ジャンル層（月 1〜2 作品、浅く多媒体）

### ターゲットジョブ（JTBD）

1. 「あれ、これ何話まで観たっけ？」を無くしたい（進捗管理）
2. 「過去に何を読んだか・観たか」を時系列で振り返りたい（ログ／資産化）
3. 気分で次の作品を選べるよう、自分の好みを俯瞰したい（傾向把握）

### 具体的なユースケース

- 深夜アニメ視聴中に「+1 話」をタップして進捗更新
- 「去年の夏に読んだ本なんだっけ」をマイライブラリで検索
- 原作マンガを読み終えたタイミングでアニメ版をチェック

## 4. Problems & Pain Points

> **✅ 実在の声**（IK さん自身の体験より取得）

### コアの問題

同じ作品を複数サービスに**二重記録**している。評価基準もサービスごとにバラバラ（10 点 vs 5 つ星）で統合不能。自分用のメモとして記録したいだけなのに、既存サービスは公開前提で他人に見られてしまう。

### 具体的な痛みポイント

1. **二度手間**: 「AniList と Filmarks 両方に記録してる」
2. **評価基準のズレ**: 「10 点満点と 5 つ星で違う」
3. **プライバシーの無さ**: 「自分用のメモとして記録したいだけなのに、勝手に他の人に見られてコメントされるのがうざい」「匿名投稿できない」「鍵垢にできない」

### 代替手段の限界

- 各単一ジャンル特化サービス → 横断できない
- Notion テンプレ・スプレッドシート → 手動管理、作品メタ無し、続かない
- 「記録しない」 → 忘却、資産にならない

### 何をコストさせているか

- 同じ記録作業を 2 回以上 → 時間の無駄
- 評価の一元比較ができない → 過去の蓄積が資産にならない
- 見せたくない記録まで公開される → 記録そのものを躊躇する

### 感情面

- 「また記録するの面倒」
- 「見られたくないのに見られてしまう気まずさ」
- 「時間を使ったのに何も残っていない気がする」という虚無感

## 5. Competitive Landscape

| 区分 | 競合 | 足りないところ |
|---|---|---|
| **直接（日本）** | Annict、Filmarks、読書メーター、ブクログ | **単一ジャンル特化**。多ジャンル記録はできない |
| **直接（海外）** | AniList、MyAnimeList、Letterboxd、Goodreads | 単一ジャンル。日本語 UI 弱い |
| **セカンダリ** | Notion テンプレ、Google スプレッドシート | 手動管理、作品メタデータ無し、続かない |
| **間接** | 「記録しない・頭に任せる」 | 忘却、資産にならない |

### 全競合に共通する弱点

- **ジャンル横断の一元化ができない**（単一ジャンル特化のため）
- **プライベート記録機能が弱いか存在しない**（多くは公開前提） → Phase 2 でここを差別化に追加予定

## 6. Differentiation

### Phase 1 — 今すぐ訴求できる差別化点（実装済み）

1. **6 ジャンル横断の一元管理**
   - アニメ・映画・ドラマ・本・漫画・ゲームを 1 アプリで
   - 既存サービスは単一ジャンル特化
2. **統一された評価基準（全て 10 点満点）**
   - 「複数サービスの評価バラバラ問題」を解決
3. **ワンクリック進捗更新**
   - 視聴/読書中に即記録できる（+1 話 / +1 巻 / 観た / 読了 / クリア）
4. **クリーンなエディトリアル UI**
   - ダークなオタク UI を避けた雑誌的な読み物体験

### Phase 2 — 機能追加後に訴求する差別化点

1. **メディアごとに異なる好みを反映したクロスメディアおすすめ**
   - 「アニメはアクションが好きだが本はサスペンスが好き」のように媒体別の嗜好を学習して推薦
   - **現状は全記録を 1 プロファイルに統合する簡易実装**。per-media 実装への強化を予定
2. **非公開（鍵垢）モード**
   - 「自分だけのための記録ツール」としての訴求を追加
   - 現状は公開前提設計。非公開オプション機能を must-have として追加予定

## 7. Objections & Anti-Personas

### 予想される反論と返し方

| 反論 | 返し方 |
|---|---|
| 「Annict の方が機能が細かい」 | ジャンル特化が欲しいならそう。Recolly は**全部まとめたい人**向け |
| 「データ移行が面倒」 | 現状エクスポート/インポート未対応（**将来の検討事項**） |
| 「また新しいサービス使うの？」 | 複数サービスを**減らす**ためのサービスという逆転の訴求 |
| 「Filmarks みたいに他の人の感想も見たい」 | 公開プロフィール・コミュニティ機能は搭載（Phase 1 は公開ベース）。将来的に鍵垢も選べる |

### アンチペルソナ（Recolly が合わない人）

- 1 ジャンルしか消費しない人（アニメだけ、本だけ）
- ランキング・ソーシャル機能が主目的の人
- 既存サービスに深く根を張っていて、コミュニティごと動く気がない人

## 8. Switching Dynamics（JTBD 四力）

- **Push**（今の状況の不満）: 複数サービス管理疲れ、二度手間、評価基準のズレ
- **Pull**（Recolly への魅力）: 1 アプリ統合、10 点統一、シンプル UI、（Phase 2 で）鍵垢対応
- **Habit**（既存サービスへの慣れ）: 長年の記録の蓄積、フレンド関係、UI 習熟
- **Anxiety**（移行への不安）: 過去データ移行できない、サービスが潰れないか、他人に見られないか

→ **Push + Pull > Habit + Anxiety** でないと乗り換えない。ここが最大の壁。

## 10. Brand Voice

- **トーン**: 静か、上品、落ち着き
- **スタイル**: エディトリアル（雑誌の読み物のような文体）、絵文字を使わない
- **パーソナリティ**: 「物静かな本好きの友人」「古書店の店主」「大人向け文芸誌」
- **書き方のルール**:
  - 興奮させるより、静かに腹落ちさせる
  - 機能の自慢より、ユーザーの生活の変化を語る
  - 「！」の多用禁止
  - 「最強」「革命」「神」のような強い言葉を避ける
