# ADR-0026: レスポンシブデザインに3段階ブレークポイントを採用

## ステータス
承認済み

## 背景
RecollyはPWA（ADR-0014）としてスマホでの利用を重視している。PC・タブレット・スマホそれぞれで最適なレイアウトを提供するために、画面幅に応じたレイアウト切り替え（レスポンシブデザイン）の戦略を決める必要がある。

## 選択肢

### A案: 2ブレークポイント（768px + 1024px）で3段階切り替え（採用）
- **これは何か:** 768pxと1024pxの2つの境界線で、スマホ（〜768px）/ タブレット（769〜1024px）/ PC（1025px〜）の3段階にレイアウトを切り替える方式。業界標準の値を使用
- **長所:** シンプル。ほとんどのデバイスをカバー。768pxと1024pxはiPadの縦横幅を基準にした業界標準値で情報が多い
- **短所:** 小さいスマホ（375px）や大画面（1440px+）への微調整がない

### B案: 1ブレークポイント（768pxのみ）で2段階切り替え
- **これは何か:** スマホ/PCの2段階だけ
- **長所:** 最もシンプル。管理が楽
- **短所:** タブレットでの表示が中途半端になりがち（PCと同じ表示だとNavBarが窮屈）

### C案: 4+ブレークポイント（Bootstrap方式）で6段階切り替え
- **これは何か:** xs/sm/md/lg/xl/xxlの6段階（576/768/992/1200/1400px）
- **長所:** あらゆるデバイスに最適化できる
- **短所:** 管理が複雑。CSSが膨大になる。個人プロジェクトには過剰

## 決定
A案（2ブレークポイント、3段階切り替え）を採用。

| 画面幅 | デバイス | NavBar | BottomTabBar | レイアウト |
|--------|---------|--------|-------------|-----------|
| 1025px以上 | PC | フル表示 | なし | 横並び |
| 769〜1024px | タブレット | 縮小表示 | なし | 横並び |
| 768px以下 | スマホ | ロゴのみ | 画面下に固定表示 | 縦並び |

ブレークポイント値はCSS変数で定義（`tokens.css`）:
```
--breakpoint-mobile-max: 768px
--breakpoint-tablet-max: 1024px
```

## 理由
- **PC・タブレット・スマホそれぞれで最適な表示にしたかった。** 2段階（B案）だとタブレットの表示が中途半端になる。3段階ならデバイスごとに適切なレイアウトを提供できる
- **業界標準のブレークポイント値を使った。** 768px（iPadの縦幅）と1024px（iPadの横幅）はBootstrapやTailwind CSS等でも採用されている標準値。独自の値を使う理由がない
- 6段階（C案）は個人プロジェクトには過剰。3段階で十分なUXが実現できている

## 実装の特徴

### CSSだけで切り替え（JavaScriptを使わない）
NavBarとBottomTabBarは両方HTMLに存在し、CSSの `display: none/flex` で表示を切り替える。JavaScriptで画面幅を判定する処理は不要。

### タッチターゲット44px以上
Appleの推奨値に従い、スマホでタップ可能な要素は最小44pxの高さを確保。BottomTabBarのタブは52px。

### ノッチ対応
iPhone等の画面下部の安全領域を考慮し、BottomTabBarに `env(safe-area-inset-bottom)` で余白を追加。

### 各ページのレスポンシブ対応パターン
- **コンテナ幅:** PCではmax-width（400〜800px）で中央寄せ → スマホではmax-width: 100%で全幅
- **レイアウト方向:** PCでは横並び（flex） → スマホでは縦並び（flex-direction: column）
- **グリッド列数:** PCでは4列 → スマホでは2列または1列
- **画像サイズ:** PCでは大きめ → スマホでは縮小

## 影響
- 全ページ・コンポーネントが768pxと1024pxのブレークポイントに従う
- 新しいページやコンポーネントを追加する際は、この2つのブレークポイントでのレイアウトを必ず考慮する
- ブレークポイント値の変更が必要な場合は `tokens.css` の変数を更新し、全CSSファイルのメディアクエリも合わせて修正する（CSSのメディアクエリではCSS変数を直接使えないため）
