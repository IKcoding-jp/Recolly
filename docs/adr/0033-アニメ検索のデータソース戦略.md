# ADR-0033: アニメ検索のデータソース戦略

## ステータス
承認済み

## 背景
Recollyの作品検索機能で、アニメの検索結果に2つの問題があった。

1. **TMDB（映画・ドラマのDB）がアニメをTV番組として返す**: TMDBは映画/TV番組の総合データベースで、日本のアニメもTV番組（`media_type: 'tv'`）として扱う。Recollyではこれを全て「ドラマ」にマッピングしていたため、「けいおん！」が「ドラマ」と表示されてしまっていた。
2. **AniList（アニメ専門DB）の説明が英語**: AniListは英語圏のアニメデータベースのため、作品説明（description）が英語で返る。Recollyは日本向けプロダクトなので、日本語で表示したい。

この2つを同時に解決するデータソース戦略を決める必要があった。

## 選択肢

### A案: TMDBでアニメも取得する（AniListの代わりに）
- **これは何か:** TMDBの検索結果から日本のアニメを検出し、「ドラマ」ではなく「アニメ」としてマッピングする方式。AniListはアニメ検索に使わず、漫画専用にする。
- **長所:**
  - シンプル。追加のAPI呼び出し不要
  - TMDBは `language: 'ja'` で日本語説明を返してくれるので、翻訳の問題がない
  - 実装が簡単（数行の変更）
- **短所:**
  - TMDBはアニメ専門DBではないため、マイナーなアニメが見つからない可能性がある
  - アニメ固有のメタデータ（エピソード数、放送状態、シーズン情報）の精度がAniListに劣る

### B案: AniListで検索 + TMDBから日本語説明を補完
- **これは何か:** AniList（アニメ専門DB）で検索して充実したアニメ情報を取得しつつ、説明文だけTMDBから日本語版を取得する方式。2つのAPIの長所を組み合わせる。
- **長所:**
  - AniListのアニメ網羅性と詳細メタデータが使える
  - 日本語の説明も得られる（TMDBから補完）
- **短所:**
  - 検索が遅くなる（AniList検索結果1件ごとにTMDB検索が必要）
  - タイトルの表記揺れでTMDBマッチに失敗する場合がある
  - TMDBにないアニメは英語説明のまま（フォールバック）

## 決定
**B案: AniListで検索 + TMDBから日本語説明を補完** を採用する。

## 理由
- Recollyはアニメ・漫画・ゲームなどの「物語性のあるメディア」を扱うプロダクトであり、アニメの品揃えと情報の質が重要。TMDBのアニメカバレッジでは不十分になるリスクがあった。
- 検索速度の低下は、30分間のキャッシュにより初回検索以降は問題にならない。
- TMDBマッチ失敗時は英語説明をフォールバックとして使うため、表示が完全に壊れることはない。
- A案を選ばなかった理由: TMDBのアニメデータの網羅性と精度がAniListに劣るため、アニメファン向けプロダクトとしてデータ品質を優先した。

## 影響
- AniListの検索結果ごとにTMDB API呼び出しが追加されるため、初回検索時のレスポンスが遅くなる。今後パフォーマンスが問題になった場合、並列リクエストの実装を検討する。
- TMDBから説明を取得するため、AniListの英語/ローマ字タイトルをメタデータに保持する設計になった。タイトルの表記揺れによるマッチ失敗が発生した場合、マッチングロジックの改善が必要になる。
