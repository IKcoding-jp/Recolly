# ADR-0024: Ruby 3.3とNode.js 22を採用

## ステータス
承認済み

## 背景
Recollyのバックエンド（Rails）とフロントエンド（React）を動かすランタイムのバージョンを決める必要がある。新しすぎるとライブラリが未対応、古すぎるとセキュリティサポートが切れるため、安定した最新版を選ぶ必要がある。

## 選択肢

### Ruby

#### A案: Ruby 3.3（採用）
- **これは何か:** 2023年12月リリース。YJITコンパイラ（Rubyコードを高速実行するための仕組み）の改善で性能向上。Rails 8が推奨するバージョン
- **長所:** プロジェクト開始時点の最新安定版。Rails 8の推奨。セキュリティサポート中
- **短所:** 特になし

#### B案: Ruby 3.2
- **これは何か:** 1つ前の安定版。メンテナンスモード（セキュリティ修正のみ）
- **長所:** 最も安定。全gemが対応済み
- **短所:** 新機能なし。サポート終了が近い

#### C案: Ruby 3.4
- **これは何か:** 2024年12月リリースの最新版
- **長所:** 最新機能
- **短所:** プロジェクト開始時点ではリリース直後でgemの対応が追いついていない場合がある

### Node.js

#### A案: Node.js 22（採用）
- **これは何か:** 2024年4月リリースのLTS（Long Term Support = 長期サポート版）。2027年4月までセキュリティ修正が提供される。Node.jsは偶数番号（18, 20, 22...）がLTSで、本番利用にはLTSを選ぶのが原則
- **長所:** プロジェクト開始時点の最新LTS。サポート期間が長い（2027年4月まで）
- **短所:** 特になし

#### B案: Node.js 20
- **これは何か:** 1つ前のLTS。2026年4月までサポート
- **長所:** 安定。全パッケージが対応済み
- **短所:** サポート終了が近い

## 決定
Ruby 3.3とNode.js 22を採用。プロジェクト開始時点の最新安定版（LTS）を選択。

バージョンは以下の箇所で統一管理:

| 場所 | Ruby | Node |
|------|------|------|
| Dockerfile | `ruby:3.3-slim` | `node:22-slim` |
| CI（GitHub Actions） | `ruby-version: '3.3'` | `node-version: '22'` |
| バージョンファイル | `.ruby-version: ruby-3.3.10` | — |

## 理由
- **プロジェクト開始時点の最新安定版を選んだ。** 特別な理由があるわけではなく、最新かつ安定しているバージョンを選ぶのが自然な判断
- Ruby 3.3はRails 8の推奨バージョン。Node.js 22は最新のLTSでサポート期間が長い
- 古いバージョンを選ぶとサポート終了が早く、セキュリティリスクになる

## 影響
- Ruby/Nodeのメジャーバージョンアップ時はDockerfile、CI設定、.ruby-versionを全て更新する必要がある
- マイナーバージョン（3.3.x）のアップデートはDockerイメージの再ビルドで自動適用される
