# ADR-0039: 書籍データ補完APIに openBD を採用

## ステータス
承認済み

## 背景

ADR-0038 で決めたアプローチ C-2 では、Google Books API で欠損している書籍の画像・説明を補完するために追加の外部APIを使う。日本語書籍のメタデータを提供する候補APIは複数あるため、どれを採用するかを判断する必要がある。

本件の補完は**キーワード検索ではなく ISBN ベースの補完**という用途に限定される（検索自体は引き続きGoogle Booksで行う）。そのため、キーワード検索機能の有無は必須要件ではなく、ISBNから書誌データを引けることと、無料・運用コストの低さが重要になる。

## 選択肢

### A案: openBD
- **これは何か:** 出版社・書店・図書館業界が共同運営する日本書籍メタデータの公開データベース。ISBNを指定して書誌情報・書影（カバー画像）・内容紹介などを取得できる無料API。https://openbd.jp/
- **長所:**
  - **無料かつAPIキー・登録不要**。初学者プロジェクトにとって導入ハードルが極めて低い
  - 日本書籍に特化したカバレッジ
  - ISBN から一括取得が可能（パフォーマンス良好）
  - シンプルなREST API
  - 商用利用可能
- **短所:**
  - **キーワード検索機能がない**（ISBNベースのみ）。ただし本件の用途では問題にならない
  - 日本書籍のみ（洋書カバーは期待できない）
  - 内容紹介の詳細度は出版社によってムラがある

### B案: 楽天ブックスAPI
- **これは何か:** 楽天が提供する書籍情報検索API。キーワード検索・ISBN検索の両方に対応し、書影・説明・価格・評価などが取れる。
- **長所:**
  - キーワード検索可能
  - 説明文が比較的豊富
  - 書影も安定して取れる
- **短所:**
  - **APIキー登録が必須**（無料だが楽天デベロッパーズへの登録が必要）
  - レート制限あり（無料枠1秒1リクエスト）
  - 商用利用に制約がある（アフィリエイト連携が前提のケースがある）
  - 本件の「ISBN補完」用途にはオーバースペック

### C案: 国立国会図書館サーチ（NDL Search）API
- **これは何か:** 国立国会図書館が提供する日本国内出版物の書誌データベース。図書館向けの権威データソース。
- **長所:**
  - 国立図書館管理の権威データ・無料
  - 登録不要
- **短所:**
  - **SRUプロトコル**という図書館向けの特殊な問い合わせ形式で、実装コストが高い
  - 画像（書影）の提供が限定的
  - 内容紹介テキストが少ない

### D案: Amazon Product Advertising API（PA-API）
- **これは何か:** Amazonアソシエイト参加者向けの商品データAPI。書影・レビュー・説明などが豊富。
- **長所:**
  - データが豊富
- **短所:**
  - **アソシエイト登録必須**
  - **売上実績要件**があり、新規アカウントは使えない
  - 使用制限が厳しい
  - 個人開発のRecollyには事実上使えない

## 決定

**A案（openBD）** を採用する。

## 理由

- **「ISBN補完」という限定用途に最適**: 本件はキーワード検索ではなく、Google BooksからISBNを得てから欠損データを補完するフロー。openBDはまさにこの用途に向いている
- **導入ハードルが最小**: APIキー・登録・レート制限管理なしで即使える。Recollyのような個人開発プロジェクトにとって非常に重要
- **無料かつ商用利用可能**: 将来のマネタイズ方針に影響しない
- **B案（楽天ブックス）を選ばない理由**:
  - 本件の補完用途には検索機能は不要で、openBDで十分
  - APIキー登録・レート制限管理のコストが発生する
  - C-3（ADR-0038で検討）として将来追加する余地を残す。補完で足りなければ検索でも併用を検討する
- **C案（NDL Search）を選ばない理由**: SRUプロトコル実装が初学者にとってハードルが高く、画像・説明の取れ高も低い
- **D案（Amazon PA-API）を選ばない理由**: 売上要件により個人開発では実質利用不可

## 影響

- 新規: `backend/app/services/external_apis/openbd_client.rb`
- 新規: `backend/spec/services/external_apis/openbd_client_spec.rb`
- `WorkSearchService#enrich_books_via_openbd` から呼び出される
- `GoogleBooksAdapter#normalize` がISBN-13/ISBN-10をmetadataに含めるよう修正
- openBDへのHTTP呼び出しは既存の `faraday` + WebMock（テスト）で扱う
- openBDがダウンしている場合、`OpenbdClient#fetch` は例外を捕捉して nil を返す設計。既存のGoogle Books結果はそのまま表示される（フォールバック動作）
- 新しい環境変数は不要（APIキー・秘密鍵なし）
